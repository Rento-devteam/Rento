```    sequenceDiagram
    actor User as Пользователь (Инициатор)
    actor OtherUser as Другая сторона
    actor Moderator as Модератор
    participant WebApp as Веб-приложение
    participant DealService as Сервис сделок
    participant DisputeService as Сервис споров
    participant Database as База данных
    participant PaymentService as Платежный сервис
    participant PaymentGateway as Платежный Шлюз
    participant SupportService as Служба поддержки
    participant NotifyService as Сервис уведомлений
    participant ARSEngine as Модуль расчета ARS

    Note over User,DealService: Предусловие: Активная сделка, стороны не могут прийти к соглашению

    User->>WebApp: Нажатие кнопки "Открыть спор" (в сделке / чате)
    WebApp->>DealService: GET /deals/{id}/dispute/initiate
    DealService->>Database: Проверка статуса сделки
    Database-->>DealService: deal_data (status, participants, hold_info)
    
    alt Сделка не может быть переведена в спор
        DealService-->>WebApp: 409 Conflict (deal_invalid_status)
        WebApp-->>User: Уведомление: "Спор недоступен для данной сделки"
    else Сделка может быть оспорена
        DealService-->>WebApp: dispute_form_data
        WebApp-->>User: Отображение формы спора
        
        Note over WebApp: Форма спора:<br/>- Причина: повреждение / невозврат / несоответствие<br/>- Описание ситуации<br/>- Фото-доказательства
        
        User->>WebApp: Заполнение формы спора
        User->>WebApp: Прикрепление фото-доказательств
        User->>WebApp: Отправка спора
        
        WebApp->>DisputeService: POST /disputes (deal_id, reason, description, evidence_photos)
        
        DisputeService->>Database: Проверка на существующий активный спор
        Database-->>DisputeService: no_active_dispute
        
        DisputeService->>DisputeService: Создание записи спора
        DisputeService->>Database: Сохранение спора (status: "pending")
        Database-->>DisputeService: dispute_id
        
        DisputeService->>DealService: update_deal_status(deal_id, "dispute")
        DealService->>Database: Обновление статуса сделки
        Database-->>DealService: deal_status_updated
        
        DisputeService->>PaymentService: hold_funds_for_dispute(deal_id)
        PaymentService->>Database: Блокировка выплат до решения спора
        Database-->>PaymentService: funds_held
        PaymentService-->>DisputeService: funds_held_confirmed
        
        DisputeService->>SupportService: create_dispute_ticket(dispute_id, deal_id, evidence)
        SupportService->>Database: Сохранение тикета
        Database-->>SupportService: ticket_id
        
        DisputeService->>NotifyService: notify_moderators(dispute_id)
        NotifyService-->>Moderator: Уведомление: "Новый спор требует рассмотрения"
        
        DisputeService->>NotifyService: send_dispute_opened_notifications(dispute_id)
        NotifyService-->>User: "Спор открыт. Ожидайте решения модератора"
        NotifyService-->>OtherUser: "Против вас открыт спор по сделке"
        
        DisputeService-->>WebApp: 201 Created (dispute_id)
        WebApp-->>User: Перенаправление на страницу спора
    end
    
    rect rgb(240, 248, 255)
        Note over Moderator,SupportService: Процесс рассмотрения спора модератором
        
        Moderator->>SupportService: Вход в панель модерации
        SupportService->>Database: Запрос списка активных споров
        Database-->>SupportService: pending_disputes
        SupportService-->>Moderator: Отображение списка споров
        
        Moderator->>SupportService: Выбор спора для рассмотрения
        SupportService->>Database: Получение полных данных спора
        Database-->>SupportService: dispute_data, deal_data, evidence_photos
        
        SupportService->>Database: Получение истории переписки
        Database-->>SupportService: chat_history
        
        SupportService->>Database: Получение фото передачи и возврата (UC-14, UC-15)
        Database-->>SupportService: handover_photos, return_photos
        
        SupportService-->>Moderator: Отображение всех материалов:<br/>- История переписки<br/>- Фото передачи<br/>- Фото возврата<br/>- Доказательства инициатора
        
        alt Недостаточно доказательств
            Moderator->>SupportService: Запрос дополнительной информации
            SupportService->>NotifyService: request_additional_evidence(dispute_id)
            NotifyService-->>User: "Модератор запросил дополнительные доказательства"
            NotifyService-->>OtherUser: "Модератор запросил дополнительную информацию"
            
            User->>WebApp: Предоставление дополнительных фото/комментариев
            WebApp->>DisputeService: POST /disputes/{id}/evidence (photos, comments)
            DisputeService->>Database: Сохранение дополнительных доказательств
            Database-->>DisputeService: evidence_saved
            DisputeService->>SupportService: notify_new_evidence(dispute_id)
            SupportService-->>Moderator: Обновленные материалы
        end
        
        Moderator->>SupportService: Принятие решения
        
        alt Решение в пользу инициатора
            Moderator->>SupportService: resolve_dispute(dispute_id, "in_favor_of_initiator", resolution_details)
            Note over SupportService: Полный возврат залога арендатору<br/>или компенсация арендодателю
        else Решение в пользу другой стороны
            Moderator->>SupportService: resolve_dispute(dispute_id, "in_favor_of_respondent", resolution_details)
            Note over SupportService: Средства распределяются в пользу ответчика
        else Компромиссное решение
            Moderator->>SupportService: resolve_dispute(dispute_id, "compromise", resolution_details, split_amounts)
            Note over SupportService: Частичная компенсация каждой стороне
        end
        
        SupportService->>DisputeService: resolve_dispute(dispute_id, resolution, resolution_details)
        DisputeService->>Database: Обновление статуса спора (status: "resolved")
        Database-->>DisputeService: dispute_resolved
        
        DisputeService->>PaymentService: execute_dispute_settlement(deal_id, resolution)
        PaymentService->>PaymentGateway: process_dispute_payment(hold_id, settlement_amounts)
        PaymentGateway->>PaymentGateway: Распределение средств согласно решению
        PaymentGateway-->>PaymentService: settlement_complete
        PaymentService-->>DisputeService: funds_distributed
        
        DisputeService->>DealService: update_deal_status(deal_id, "completed")
        DealService->>Database: Обновление статуса сделки
        Database-->>DealService: deal_closed
        
        DisputeService->>NotifyService: send_dispute_resolution_notifications(dispute_id, resolution)
        NotifyService-->>User: Уведомление о решении спора
        NotifyService-->>OtherUser: Уведомление о решении спора
    end
    
    rect rgb(255, 248, 240)
        Note over DisputeService,ARSEngine: Альтернативный сценарий: Ложный спор (7a)
        
        alt Решение: спор признан ложным
            Moderator->>SupportService: resolve_dispute(dispute_id, "false_dispute", "Инициатор подал ложный спор")
            SupportService->>DisputeService: resolve_dispute(dispute_id, "false_dispute")
            
            DisputeService->>Database: Обновление статуса спора
            Database-->>DisputeService: dispute_closed
            
            DisputeService->>ARSEngine: report_false_dispute(initiator_id)
            Note over ARSEngine: Негативное влияние на ARS инициатора:<br/>- Снижение индекса надежности<br/>- Отметка в профиле
            
            ARSEngine->>Database: Обновление ARS инициатора
            Database-->>ARSEngine: ars_updated
            
            DisputeService->>NotifyService: send_false_dispute_notification(initiator_id)
            NotifyService-->>User: "Ваш спор признан необоснованным. Это может повлиять на ваш индекс надежности"
            NotifyService-->>OtherUser: "Спор закрыт в вашу пользу"
            
            DisputeService->>PaymentService: release_funds_original(deal_id)
            PaymentService->>PaymentGateway: возврат к стандартному сценарию выплат
        end
    end
    
    rect rgb(240, 255, 240)
        Note over DisputeService,Database: Статусы спора
        
        Database->>Database: Статусы:<br/>
        Note over Database: pending - ожидает рассмотрения<br/>
        Note over Database: investigating - модератор изучает<br/>
        Note over Database: resolved - решение принято<br/>
        Note over Database: closed - спор закрыт
    end
    
    rect rgb(240, 248, 255)
        Note over DisputeService,Database: Тикеты спора
        
        SupportService->>Database: Сохранение тикета:<br/>- dispute_id, deal_id<br/>- initiator_id, respondent_id<br/>- reason, description<br/>- evidence_photos<br/>- moderator_id (назначенный)<br/>- resolution, resolution_details<br/>- timestamps
    end
```
