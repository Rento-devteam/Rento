    sequenceDiagram
    actor User as Зарегистрированный Пользователь
    participant WebApp as Веб-приложение
    participant AuthService as Сервис аутентификации
    participant Database as База данных
    participant ESIA as ЕСИА (Госуслуги)
    participant Support as Служба поддержки

    User->>WebApp: Переход в "Настройки профиля" / "Безопасность"
    WebApp->>User: Отображение настроек
    User->>WebApp: Выбор пункта "Подтвердить личность (ЕСИА)"
    
    WebApp->>AuthService: POST /verify/esia/initiate
    AuthService->>Database: Проверка текущего статуса верификации
    Database-->>AuthService: Статус: не верифицирован
    
    alt Уже верифицирован
        AuthService-->>WebApp: 400 Bad Request (уже верифицирован)
        WebApp-->>User: Уведомление: "Вы уже прошли верификацию"
    else Не верифицирован
        AuthService-->>WebApp: 200 OK (redirect_url)
        WebApp-->>User: Перенаправление на страницу авторизации ЕСИА
        User->>ESIA: Переход на портал ЕСИА
        
        ESIA-->>User: Отображение формы авторизации
        User->>ESIA: Ввод учетных данных
        
        ESIA-->>User: Запрос согласия на передачу данных
        Note over ESIA: ФИО, дата рождения, документ
        
        alt Отказ пользователя
            User->>ESIA: Отказ от передачи данных
            ESIA-->>WebApp: Перенаправление с отказом
            WebApp->>AuthService: GET /verify/esia/callback?error=access_denied
            AuthService-->>WebApp: Отмена верификации
            WebApp-->>User: Сообщение: "Верификация отменена"
        else Согласие пользователя
            User->>ESIA: Подтверждение согласия
            ESIA->>ESIA: Генерация защищенного токена
            ESIA-->>WebApp: Перенаправление с токеном
            WebApp->>AuthService: GET /verify/esia/callback?code=token
            
            AuthService->>ESIA: Запрос данных по токену
            ESIA-->>AuthService: Передача подтвержденных данных (ФИО, дата рождения, документ)
            
            AuthService->>Database: Получение данных профиля пользователя
            Database-->>AuthService: Данные профиля (если заполнены)
            
            AuthService->>AuthService: Сопоставление данных ЕСИА с профилем
            
            alt Данные совпадают
                AuthService->>Database: Обновление статуса пользователя
                Database-->>AuthService: Статус обновлен: "Верифицирован"
                Database-->>AuthService: Бейдж добавлен
                
                AuthService->>AuthService: Пересчет индекса надежности (ARS)
                AuthService->>Database: Сохранение нового значения ARS
                Database-->>AuthService: ARS обновлен
                
                AuthService-->>WebApp: 200 OK (статус верифицирован, ARS)
                WebApp-->>User: Отображение бейджа "Проверенный пользователь"
                WebApp-->>User: Уведомление о повышении индекса ARS
                
            else Несовпадение данных
                AuthService-->>WebApp: 409 Conflict (несоответствие данных)
                WebApp-->>User: Уведомление: "Обнаружено несоответствие данных. Требуется ручная проверка"
                WebApp->>AuthService: POST /verify/esia/escalate
                AuthService->>Support: Создание обращения в службу поддержки
                Support-->>AuthService: Статус: ожидает ручной проверки
                AuthService->>Database: Обновление статуса: "Ожидает проверки"
                AuthService-->>WebApp: Обращение создано
                WebApp-->>User: Сообщение: "Данные отправлены на ручную проверку. Ожидайте уведомления"
                
                Note over Support,User: Служба поддержки проводит проверку
                Support->>AuthService: Подтверждение/отклонение верификации
                AuthService->>Database: Обновление статуса
                AuthService-->>User: Уведомление о результате проверки
            end
        end
    end
