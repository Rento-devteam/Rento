    sequenceDiagram
    actor Landlord as Арендодатель
    actor Renter as Арендатор
    participant WebApp as Веб-приложение
    participant DealService as Сервис сделок
    participant Database as База данных
    participant PhotoService as Сервис фото
    participant CameraInterface as Интерфейс камеры
    participant SupportService as Служба поддержки
    participant NotifyService as Сервис уведомлений

    Note over Landlord,DealService: Предусловие: Сделка со статусом "Ожидает передачи"

    Landlord->>WebApp: Открытие сделки
    Landlord->>WebApp: Нажатие "Начать передачу"
    
    WebApp->>DealService: POST /deals/{id}/handover/start
    DealService->>Database: Проверка статуса сделки
    Database-->>DealService: status: "awaiting_handover"
    
    DealService->>DealService: Активация чек-листа передачи
    DealService-->>WebApp: handover_session_id, checklist
    
    WebApp-->>Landlord: Отображение чек-листа передачи<br/>и интерфейса камеры
    
    loop По каждому пункту чек-листа
        Landlord->>CameraInterface: Фотографирование вещи
        Note over CameraInterface: Пункты чек-листа:<br/>- Общий план вещи<br/>- Существующие дефекты<br/>- Серийный номер/маркировка<br/>- Комплектация
        
        CameraInterface->>PhotoService: upload_photo(photo_data, deal_id, checklist_item)
        PhotoService->>PhotoService: Сохранение фото с метаданными<br/>(время, геолокация, тип)
        PhotoService->>Database: Сохранение привязки фото к сделке
        Database-->>PhotoService: photo_id сохранен
        PhotoService-->>CameraInterface: подтверждение загрузки
        CameraInterface-->>Landlord: Отображение превью фото
    end
    
    Landlord->>WebApp: Завершение фотофиксации
    WebApp->>DealService: POST /deals/{id}/handover/photos-complete
    DealService->>Database: Сохранение статуса: фото переданы
    Database-->>DealService: photos_saved
    
    DealService-->>WebApp: Фотофиксация завершена
    WebApp-->>Landlord: Уведомление: "Фотоотчет сохранен. Передайте вещь арендатору"
    
    Note over Landlord,Renter: Физическая передача вещи
    
    Renter->>WebApp: Открытие сделки
    WebApp->>DealService: GET /deals/{id}/handover/photos
    DealService->>Database: Запрос фотоотчета
    Database-->>DealService: handover_photos
    DealService-->>WebApp: photos_data
    WebApp-->>Renter: Отображение фотоотчета<br/>с возможностью подтверждения
    
    Renter->>WebApp: Нажатие "Подтвердить получение"
    
    WebApp->>DealService: POST /deals/{id}/handover/confirm
    DealService->>Database: Проверка статуса и наличия фото
    
    alt Фотоотчет отсутствует
        Database-->>DealService: no_photos
        DealService-->>WebApp: 400 Bad Request
        WebApp-->>Renter: "Фотоотчет еще не создан арендодателем"
    else Фотоотчет существует
        DealService->>Database: Обновление статуса сделки
        Database-->>DealService: status: "active_rental"
        
        DealService->>NotifyService: send_notifications(deal_id, "handover_confirmed")
        NotifyService-->>Landlord: Уведомление: "Арендатор подтвердил получение вещи"
        NotifyService-->>Renter: Уведомление: "Аренда началась. Вещь у вас"
        
        DealService-->>WebApp: 200 OK (status: "active_rental")
        WebApp-->>Renter: Перенаправление на страницу активной аренды
        WebApp-->>Renter: Уведомление: "Получение подтверждено. Аренда активна"
    end
    
    rect rgb(255, 248, 240)
        Note over Renter,SupportService: Альтернативный сценарий: Несогласие с состоянием
        
        Renter->>WebApp: Нажатие "Зафиксировать разногласие"
        WebApp->>DealService: POST /deals/{id}/handover/dispute
        
        DealService->>Database: Проверка наличия фотоотчета
        Database-->>DealService: handover_photos_exist
        
        DealService->>Database: Обновление статуса сделки
        Database-->>DealService: status: "dispute"
        
        DealService->>SupportService: create_dispute_ticket(deal_id, reason, photos)
        SupportService-->>DealService: ticket_id
        
        DealService->>NotifyService: send_dispute_notifications(deal_id)
        NotifyService-->>Landlord: Уведомление: "Арендатор зафиксировал разногласие по состоянию вещи"
        NotifyService-->>Renter: Уведомление: "Разногласие зафиксировано. Ожидайте решения службы поддержки"
        
        DealService-->>WebApp: 200 OK (status: "dispute", ticket_id)
        WebApp-->>Renter: Уведомление: "Сделка переведена в статус 'Спор'. Поддержка будет связываться с вами"
        
        Note over SupportService,Landlord: Служба поддержки анализирует фото<br/>и принимает решение
        SupportService->>DealService: resolve_dispute(deal_id, resolution)
        DealService->>Database: Обновление статуса (active_rental / cancelled)
    end
    
    rect rgb(240, 248, 255)
        Note over DealService,Database: Структура чек-листа передачи
        
        Database->>Database: Чек-лист по умолчанию:<br/>
        Note over Database: 1. Общий план вещи<br/>
        Note over Database: 2. Фото дефектов (при наличии)<br/>
        Note over Database: 3. Серийный номер / маркировка<br/>
        Note over Database: 4. Комплектация (документы, аксессуары)<br/>
        Note over Database: 5. Упаковка (опционально)
    end
    
    rect rgb(240, 255, 240)
        Note over Landlord,Renter: Дополнительно: Отказ от приема
        
        Renter->>WebApp: Нажатие "Отказаться от получения"
        WebApp->>DealService: POST /deals/{id}/handover/reject
        DealService->>Database: Проверка фотоотчета
        DealService->>PaymentService: release_hold(deal_id)
        Note over PaymentService: Снятие блокировки средств
        DealService->>Database: Обновление статуса: "cancelled"
        DealService->>NotifyService: уведомление обеих сторон
        WebApp-->>Renter: "Вы отказались от получения. Сделка отменена"
    end
