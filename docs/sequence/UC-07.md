**MVP:** после `POST /listings/{id}/publish` объявление переходит в `ACTIVE` без очереди модерации. Значение `PENDING_MODERATION`, эндпоинты `/moderation/*` и сценарии панели модерации (например UC-21) относятся к **будущему этапу**, не к MVP.

```mermaid
    sequenceDiagram
    actor User as Арендодатель
    participant WebApp as Веб-приложение
    participant UploadService as Сервис загрузки
    participant ValidationService as Сервис валидации
    participant Storage as Хранилище (S3/CDN)
    participant Database as База данных

    Note over WebApp,Database: Предусловие: Карточка вещи существует, фото < 10

    User->>WebApp: Переход к редактированию карточки / добавлению фото
    WebApp->>ListingService: GET /listings/{id}/photos
    ListingService-->>WebApp: Текущее количество фото (count)
    WebApp-->>User: Отображение интерфейса загрузки<br/>Индикатор: X/10 фото

    loop Загрузка каждого файла (до 10)
        User->>WebApp: Выбор файла(ов) на устройстве
        
        alt Превышение лимита (11-е фото)
            WebApp-->>User: Блокировка выбора<br/>Сообщение: "Достигнут лимит в 10 фотографий"
        else Лимит не превышен
            WebApp->>UploadService: POST /upload (file, listing_id)
            
            UploadService->>ValidationService: validate_file(file)
            
            alt Неподходящий формат
                ValidationService-->>UploadService: Ошибка: неподдерживаемый формат
                UploadService-->>WebApp: 415 Unsupported Media Type
                WebApp-->>User: Уведомление: "Поддерживаются только JPEG, PNG, WebP"
            else Файл слишком большой
                ValidationService-->>UploadService: Ошибка: превышен размер
                UploadService-->>WebApp: 413 Payload Too Large
                WebApp-->>User: Уведомление: "Файл превышает максимальный размер (10MB)"
            else Файл валидный
                ValidationService-->>UploadService: Файл прошел проверку
                
                UploadService->>UploadService: Генерация уникального имени файла
                
                WebApp-->>User: Отображение прогресса загрузки (0-100%)
                
                UploadService->>Storage: upload_file(file, generated_name)
                
                alt Ошибка загрузки на сервер
                    Storage-->>UploadService: Ошибка сети/сервера
                    UploadService-->>WebApp: 500 Internal Server Error
                    WebApp-->>User: Уведомление: "Сбой загрузки. Повторить попытку?"
                    User->>WebApp: Повторная попытка загрузки
                else Успешная загрузка
                    Storage-->>UploadService: URL файла
                    
                    UploadService->>Database: save_photo_metadata(listing_id, url, order)
                    Database-->>UploadService: Метаданные сохранены
                    
                    UploadService->>UploadService: Генерация превью
                    UploadService-->>WebApp: 200 OK (photo_data, preview_url)
                    
                    WebApp->>WebApp: Обновление счетчика (+1)
                    WebApp-->>User: Отображение превью изображения<br/>Индикатор: X/10 фото
                end
            end
        end
    end

    User->>WebApp: Нажатие кнопки "Сохранить" / "Опубликовать"
    
    WebApp->>ListingService: POST /listings/{id}/publish
    ListingService->>Database: Проверка количества фото
    
    alt Фото отсутствуют
        Database-->>ListingService: photo_count = 0
        ListingService-->>WebApp: 400 Bad Request
        WebApp-->>User: Уведомление: "Необходимо загрузить хотя бы одно фото"
    else Фото загружены
        Database-->>ListingService: photo_count > 0
        ListingService->>Database: Обновление статуса объявления
        Database-->>ListingService: Статус обновлен до "ACTIVE"
        ListingService-->>WebApp: Объявление опубликовано
        WebApp-->>User: Перенаправление в карточку вещи<br/>Уведомление: "Объявление опубликовано"
    end
    
    rect rgb(240, 248, 255)
        Note over User,Storage: Альтернативный сценарий: Drag-and-Drop
        User->>WebApp: Перетаскивание файлов в область загрузки
        WebApp->>WebApp: Обработка drop-события
        WebApp->>UploadService: Асинхронная загрузка файлов
        Note over WebApp: Тот же процесс валидации и загрузки
    end

```
